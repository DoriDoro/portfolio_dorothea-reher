003 : Chess Tournament

Python - GitHub - menu-navigation - TinyDB

In this project we had to 'structure the code of a Python program using a design pattern', 'writing rebust Python code using PEP 8' and 'use object-oriented programming to develop a Python program'.

In this role, I was a junior developer, freelancing to write simple scripts. For a chess club I had the possibility to write a program to organise the Chess Tournaments with a software. I write a Tournament Management Tool to help the club to play the Chess Tournaments offline.

The user should be able to create a player and create a tournament with eight players. The user can navigate in main menu to start a tournament with 4 rounds. The first pairing of the players is randomly. For the next rounds the player should not have already played against each other. At the end of each round the pair results and the score of each player is displayed. After the end of a tournament the end results of all tournaments are displayed.

Learning experience?

I had to follow the Model-View-Controller (MVC) design and create classes to act as templates for the tournament and players. To display the standings, pairings and other statistics as results I used views. Write controllers to accept user data, produce match results, launch new tournaments, etc.

At his point the MVC design was not easy, and sometimes it is difficult to know which function/method belongs where.

This project I realised the first time a menu navigation in the console.
