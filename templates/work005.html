005 : AlgoInvest_Trade

Python - Algorithm - Pandas - Itertools - GitHub

This algorithm project has three parts. The first part was to create a brute force algorithm. The second part an optimization of the brute force algorithm and the third part included two additional datasets. For the third part we had to clean the dataset.

The algorithms have to maximize the profit made by the client after two years of investment. We received a list of shares ['name_of_share', 'price', 'profit'] and the algorithms have to suggest the most profitable stocks that the client should buy to maximize their profit after two years.

For these algorithms we can use each share just once, we can not buy a fraction of one share and we have to spend maximum 500 Euros per customer. For the brute force algorithm we received a list of 20 shares. For the optimization we have two datasets with in total 1000 shares.

The choice of share data and the budget should be done by the user/client.

Learning experience?

For the brute force algorithm to produce all possible combinations I have used Itertools library. I have created two different algorithms, one with itertools.permutations and the second with itertools.combinations. For this reason the bruteforce.py file is referring to the bruteforce_utils.py file.

I had to clean the datasets of 1000 shares before using. The dataset was checked for duplicates and cleaned negative and zero values in the price table inside the dataset. For the cleaning part I uses the Pandas library.

The aim of these algorithm project was to face one task realized with different algorithms. To get the difference between each algorithm and to understand the big-o notion. To understand the time and space capacity of each algorithm and for which situation which algorithm should be used.
