006 : ToDoro Application

Python - Django - Bootstrap - GitHub - Heroku

Single Page Application with a todo list. The todo item can be added, updated, deleted and marked completed. The list is a table which can be updated, marked completed or the item can be deleted. When a todo item will be marked as completed the checkbox will be ticked and the incomplete text will change to complete. In the navbar new todo items can be added to the list. Above the table of todo items you see all todo items listed.

Learning experience?

ToDoro is a todo app with the CRUD functionalities. There are two models and a ForeignKey relationship. A todo list should have a title and the possibility to add multiple items to a list. During the development of the todo app there were some steps that you can't see in the final version. It was a development with ups and downs. At first I wanted to use custom CSS styling, but it didn't last long, that's when I realized that using Bootstrap makes a lot more sense. In the development of the ToDoro lists, there was an uncertainty between applying a ManyToMany and a ForeignKey relationship. With the application of a ForeignKey relationship, I better understood the use of this field. To be able to display the ToDoro List I had to figure out how to do that. I looked into InlineFormset and ModelFormsets. However, I decided against it, since I did not like the representation. To be able to display the title and the items of two different models I had to deal with related objects reference. Only with this it was possible for me to display the ToDoro List. For the first time during the development I got to know the Python shell. This project was indeed a very important one for me. The beginning of a new era. The CRUD functionalities were not so easy, especially to clear the whole list. Although the result is only three lines of code. Also showing if an item is already completed or not was a small task.
