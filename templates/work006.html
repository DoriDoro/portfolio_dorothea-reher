006 : ToDo Application

Single Page Application with a todo list. The todo item can be added, updated, deleted and marked completed. The list is a table which can be updated, marked completed or the item can be deleted. When an todo item will be marked as completed the check box will be ticked and the incomplete text will change to completed. In the navbar new todo items can be added to the list. Above the table of todo items you see all todo items listed.

Learning experience?

First Django CRUD application. Created a Todo model to display the todo item on the home page. I needed, to the home page view, four additional views: delete, edit, mark complete and mark incomplete. A TodoForm was created to edit the todo item and update it on the home page. Messages are integrated to show the success or the failure of the action.



The home page is a simple representation. Just styling like different font sizes and strikethrough text are the characteristics of the display of home page.
